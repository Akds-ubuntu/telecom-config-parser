# Парсер конфигураций базовых станций
## Реализованный функционал

Проект решает две основные задачи:
1. **Генерация артефактов из модели:** Чтение исходной информационной модели из `impulse_test_input.xml` (формат XMI) и генерация файлов `meta.json` и `config.xml`.
2. **Работа с патчами конфигураций:** Сравнение двух конфигурационных файлов (`config.json` и `patched_config.json`), вычисление разницы (`delta.json`) и применение этой разницы к исходному файлу для получения итогового `res_patched_config.json`.

## Структура проекта

```text
telecom-config-parser/
├── input/                      # Директория с исходными данными
│   ├── impulse_test_input.xml  # Исходная информационная модель
│   ├── config.json             # Исходная конфигурация
│   └── patched_config.json     # Обновленная конфигурация
├── out/                        # Директория для сгенерированных артефактов (создается автоматически)
├── artifact_generator.py       # Модуль генерации XML и JSON из внутренних структур
├── config_manager.py           # Модуль вычисления и наложения дельты
├── model_parser.py             # Модуль парсинга XMI в словарь данных
├── main.py                     # Точка входа в приложение
└── README.md                   # Документация проекта
```
## Требования
* Python: версия 3.8 или выше

## Инструкция по запуску
1. Клонирование репозитория
``` bash
git clone https://github.com/Akds-ubuntu/telecom-config-parser.git
cd telecom-config-parser
```
2. Запуск скрипта
``` bash
python3 main.py
```
